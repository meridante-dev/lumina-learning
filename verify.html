Evidence Verifier

Independently verify an exported learning-evidence file. The check runs entirely in your browser — nothing is uploaded, and it does not depend on the platform that issued the file. Each event is hashed (SHA-256) together with the hash of the previous event; if any event was altered, inserted, or removed after the fact, the chain below will break.

How it works: for every event i, the verifier recomputes sha256(JSON(event minus hash)) and requires it to equal event.hash, and requires event.prevHash == hash(event i−1) (the first event chains from "genesis"). Any SHA-256 implementation can reproduce this check without this page. Server-side, events are additionally stored append-only (create-only security rules) with server-authoritative timestamps.

The Bitcoin check: if the file carries an OpenTimestamps proof, the record's head hash was committed into a Bitcoin block. This page recomputes that block's merkle root from the proof and compares it against the real blockchain (via a public explorer — a convenience, not a trust root: the same check runs against your own node). The .ots file is standard, so ots verify or opentimestamps.org will confirm it without any software of ours.

What this does and does not prove. It proves the record existed by the time of that block, so it cannot have been back-dated, and that no event was altered since. It does not prove the learning claims inside it are true — only that they are old and unedited.
